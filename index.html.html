02 — INTERIOR DE LA CPU

Las Tres Unidades Funcionales

La CPU contiene tres unidades que trabajan en coordinación permanente. Cada una tiene registros especializados y funciones precisas.

🔊 NARRACIÓN — UNIDADES INTERNAS DE LA CPU
🎛️
UNIDAD II · CU
Unidad de Control

Dirige el flujo de datos hacia los lugares adecuados. Coordina la ejecución determinando qué operaciones se hacen y cuándo, activando señales de control de forma oportuna.

PCProgram Counter — dirección siguiente instrucción
IRInstruction Register — instrucción en ejecución
MARMemory Address Register — dirección a acceder
MBRMemory Buffer Register — dato leído o a escribir
📋
UNIDAD I · RU
Unidad de Registros

Almacenamiento interno de muy alta velocidad. Los registros son similares a la memoria pero están construidos dentro de la CPU, con conexión física directa a las demás unidades.

GPRPropósito general: AX, BX, CX, DX
CCRFlags de estado: N, Z, C, V (PSW)
SPStack Pointer — cima de la pila
BPBase Pointer — base del marco de pila
➕
UNIDAD III · ALU
Aritmética y Lógica

Realiza todas las operaciones sobre datos. Recibe operandos de los GPR, procesa y devuelve resultado. Actualiza los flags del CCR. Se comunica con otros subsistemas vía bus interno.

ARITADD SUB MUL DIV — operaciones numéricas
LÓGIAND OR NOT XOR — booleanas bit a bit
DESPSHL SHR — desplazamiento de bits
CMPComparación — actualiza flags CCR

03 — CICLO DE OPERACIÓN

Ciclo Fetch · Decode · Execute

En cada ciclo de reloj la CPU ejecuta tres fases que se repiten de forma continua e infinita mientras la computadora está encendida. Haz clic en cada fase para detalles.

🔊 NARRACIÓN — CICLO FETCH-DECODE-EXECUTE
01
📥
FETCH
RECUPERACIÓN
El PC indica la dirección. Se copia al MAR. La memoria envía la instrucción al MBR. Se transfiere al IR. El PC se incrementa.
PC→MAR MEM→MBR MBR→IR PC++
02
🔍
DECODE
DECODIFICACIÓN
La CU interpreta el opcode del IR. Determina la operación y los operandos. Genera señales de control. Si hay operandos en memoria, realiza un acceso adicional.
IR→CU opcode señales ctrl
03
⚡
EXECUTE
EJECUCIÓN
La ALU opera con los GPR. El resultado vuelve a un GPR o a memoria. Los flags del CCR se actualizan. Las bifurcaciones modifican el PC.
GPR→ALU ALU opera →GPR/MEM CCR flags

04 — SECUENCIA DE ARRANQUE

Desde Energización al Programa Principal

Secuencia exacta de eventos, señales y componentes involucrados desde que se presiona el botón de encendido hasta que el SO queda en operación. Haz clic en cada fase.

🔊 NARRACIÓN — SECUENCIA DE ARRANQUE COMPLETA
FASE 01 — ENERGIZACIÓN
⚡ Suministro de Energía (PSU)
La fuente de poder convierte CA→CC. Genera voltajes regulados (+12V, +5V, +3.3V). Cuando los voltajes son estables envía la señal POWER_GOOD al motherboard, que libera la línea de RESET del microprocesador.
POWER_GOOD +12V / +5V / +3.3V → RESET liberado
FASE 02 — VECTOR DE ARRANQUE
🔄 Primer contenido del PC
Al liberarse el RESET, el PC se carga automáticamente con la dirección del Vector de Arranque (en ROM/Flash). En x86, esta es la dirección 0xFFFF0. Es la primera instrucción ejecutada por el microprocesador recién energizado. El formato del opcode determina cuántas localidades ocupa.
PC ← Vector Arranque ROM / Flash Primera instrucción
FASE 03 — POST
🔬 Power-On Self Test (BIOS/UEFI)
La BIOS ejecuta rutinas de diagnóstico: verifica la CPU, cuenta e inicializa la RAM, detecta dispositivos en el bus, inicializa la Tabla de Vectores de Interrupción (IVT/IDT), inicializa el controlador de video. Errores críticos: códigos de beep.
Test CPU/RAM Init BUS IVT activa Detectar HW
FASE 04 — BOOTLOADER
💿 Carga del Gestor de Arranque
La BIOS busca el dispositivo de arranque (según orden de boot). Lee el MBR (sector 0 del disco) o partición EFI. Lo carga en RAM y transfiere el control. El bootloader (GRUB / BOOTMGR) localiza el kernel del SO.
MBR → RAM PC ← dir RAM Bootloader exec
FASE 05 — KERNEL DEL SO
🖥️ Carga e Inicio del Sistema Operativo
El bootloader carga el kernel en RAM. El kernel inicializa drivers, configura memoria virtual, activa el planificador de procesos (scheduler), lanza el proceso init. El ciclo FDE continúa, ahora ejecutando instrucciones del SO.
Kernel en RAM IDT configurada Drivers init Scheduler activo
FASE 06 — OPERACIÓN NORMAL
✅ Programa Principal en Ejecución
El SO queda corriendo. El ciclo Fetch→Decode→Execute se repite infinitamente. Las interrupciones permiten atender eventos externos. Las subrutinas modularizan el código usando la pila. El PC siempre apunta a la siguiente instrucción a ejecutar.
Ciclo FDE ∞ Interrupciones CALL / RET (Pila) E/S activa

05 — CONCEPTOS CLAVE

Definiciones y Fundamentos

Conceptos fundamentales abordados en el módulo. Haz clic para ampliar.

🔊 NARRACIÓN — CONCEPTOS FUNDAMENTALES
0️⃣1️⃣
Sistema Binario
Las computadoras usan binario por la naturaleza de los circuitos electrónicos: dos estados físicos estables (encendido=1, apagado=0). No es para simplificar cálculos sino por la física del silicio.
🔧
Subrutinas (CALL / RET)
Bloque de código modularizado para una tarea específica. CALL guarda el PC en la Pila y salta. RET recupera el PC de la Pila y retorna. Permite anidamiento gracias al Stack LIFO.
🚨
Interrupciones
Señal que suspende la ejecución actual. La CPU guarda su estado, atiende la ISR (Interrupt Service Routine) y restaura el contexto. La IVT/IDT mapea cada interrupción a su rutina.
📝
Registros vs. Memoria RAM
Los registros son memoria interna a la CPU con acceso en 1 ciclo. La RAM es externa, más lenta y grande. Un registro es parecido a la memoria, pero su ubicación dentro del chip le da acceso directo y ultrarrápido.
🔲
Microprocesador vs. Microcontrolador
Microprocesador: solo la CPU (CU+ALU+Registros). Necesita RAM y ROM externos. Microcontrolador: CPU+RAM+ROM+periféricos en un solo chip. El microcontrolador sí contiene internamente una CPU.
🏛️
Arquitectura Von Neumann
Modelo donde instrucciones y datos comparten la misma memoria. El formato del código de operación (opcode) determina cuántas localidades de memoria ocupa cada instrucción del programa.
